# SafeOps Course

Production-grade DevOps and Site Reliability Engineering, taught through guardrails, failure patterns, and recovery discipline.

This repository contains the working materials behind the **SafeOps Academy** course.

## What this course is

SafeOps is a hands-on course focused on building and operating modern production platforms safely.

The course covers:

- Kubernetes
- Terraform
- GitOps with Flux
- CI/CD guardrails
- secrets management with SOPS
- network isolation
- workload hardening
- resource management and QoS
- availability engineering
- observability
- backup and restore validation
- controlled chaos engineering
- AI-assisted SRE inside human-owned guardrails

## Core idea

AI is a very well-read junior engineer.

It is fast, useful, and confident.

It can help with analysis, review, summarization, and investigation.

But it does not own production risk.

SafeOps teaches how to use AI without increasing blast radius, ambiguity, or unsafe automation.

## What you will learn

This course teaches engineers how to:

- reduce blast radius
- preserve rollback paths
- investigate incidents using evidence
- design systems that survive mistakes
- validate recoverability instead of assuming it
- build workflows that remain safe under pressure

## Teaching model

The course is built around realistic failure patterns.

Each major lesson focuses on:

1. a realistic mistake or incident pattern  
2. the confusing symptoms it creates  
3. the investigation path  
4. the containment or rollback path  
5. the guardrail that prevents the same class of failure

This is not a theory-only course.

It is designed to feel like real production engineering.

- structure over confidence
- guardrails over improvisation
- rollback over clever fixes
- observability before automation
- evidence before action
- containment over speed
